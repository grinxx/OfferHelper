# 使用指南

## 推荐输入

第一次使用时，不要直接说"帮我改简历"。建议提供：

- 当前身份：应届生、实习求职、转行、社招早期等；
- 目标城市；
- 目标岗位或排除岗位；
- 简历文件或简历文本；
- 3-10 个真实 JD；
- 时间限制和个人约束。

示例：

```text
$offer-helper
我是 2026 届本科生，目标城市深圳/广州，想投 AI 应用实习、产品运营实习和内容增长。
我会上传简历，并粘贴 5 个真实 JD。请先问我必要问题，再建立求职案例目录。
```

## 常见任务

### 挖掘优势

```text
请围绕我的项目、实习、社团、课程作业和自学经历追问，帮我整理 strengths.md。每个优势都要写证据、行为、能力和岗位信号。
```

### 匹配岗位

```text
这里有 5 个 JD，请整理成 jobs.jsonl，并输出 job-matches.csv。低于 60 分的岗位只放观察池，不建议投递。
```

### 优化简历

```text
基于这个目标 JD，帮我做 resume-review.md。不要编经历；缺少证据的内容标记 needs_proof。
```

### 训练面试

```text
我先发一段自我介绍，请检查结构、证据、岗位关联、废话和风险表达。一次只问我一个追问问题。
```

## 产物说明

- `profile.yaml`：基本信息、目标城市、岗位偏好和限制条件。
- `experience-assets.md`：真实经历资产库。
- `strengths.md`：优势证据链。
- `target-roles.csv`：岗位簇假设。
- `jobs.jsonl`：用户提供的岗位数据。
- `job-matches.csv`：岗位匹配评分与下一步动作。
- `resume-review.md`：简历诊断和改写建议。
- `interview-story-bank.md`：面试故事库。
- `interview-practice.md`：模拟面试记录。
- `application-tracker.csv`：投递跟踪。

## 重要提醒

OfferHelper 的输出是求职参谋材料，不是录用保证。任何投递、私信、简历提交或平台操作，都应由用户自己确认和执行。
