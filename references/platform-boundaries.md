# 平台使用边界（Platform Boundaries）

将招聘平台作为用户授权的信息来源，而非自动化操作对象。

## V1 允许的操作

- 用户从 Boss直聘、猎聘、领英、拉勾、智联、前程无忧、学校就业网、公司招聘页面或其他来源粘贴 JD 文本。
- 用户上传截图、导出文件、复制表格或手动保存的页面。
- 用户在浏览器中打开页面，请求 Agent 分析可见信息。
- 每次外部投递、私信或资料修改，均由用户手动确认后执行。

## V1 不允许的操作

- 不后台批量爬取。
- 不绕过登录、验证码、反爬机制、频率限制或平台规则。
- 不自动提交申请。
- 不自动给招聘方发消息。
- 不在 Skill 内存储账号 Cookie 或凭据。
- 未经用户提供已验证的集成凭据和使用条款，不声称拥有平台授权或官方 API 访问权限。

## 推荐表达

使用：

- "监督式精准投递助手"
- "岗位整理与匹配"
- "生成投递准备清单"
- "用户确认后手动投递"

避免：

- "全自动海投"
- "自动刷岗位"
- "保证面试"
- "绕过平台限制"

## 来源记录字段

每条岗位信息都需记录：

- platform（平台）
- company（公司）
- title（职位名）
- url（链接，如有）
- observed_at（查看日期）
- source_type: `user_paste` | `screenshot` | `csv_export` | `browser_visible` | `manual_entry`
- JD 原文或摘要
