# 简历优化标准（Resume Rubric）

通过让真实证据在目标岗位面前清晰可读来优化简历。不编造，不夸大。

## 简历审查顺序

1. 阅读目标 JD，提取关键词、硬性要求和隐含能力。
2. 阅读 `experience-assets.md` 和 `strengths.md`。
3. 将每个重要 JD 要求映射到用户的真实证据。
4. 在改写前先识别缺失的证据。
5. 用真实动作、方法、结果和岗位关键词提出改写建议。

## 条目写法

推荐格式：

```text
在 <场景/项目> 中，为解决 <问题>，使用 <方法/工具> 完成 <动作>，产出 <结果/证据>，体现 <岗位关键词能力>。
```

简版：

```text
使用 <工具/方法> 完成 <动作/产出>，使 <结果/指标/证据>，支撑 <岗位关键词>。
```

## 审查标签（Labels）

在审查输出中使用以下标签：

- `use_as_is`：可直接使用，稍作润色即可。
- `rewrite`：有真实证据但表达偏弱，需要改写。
- `needs_proof`：说法有吸引力但缺乏证据。
- `remove`：无支撑、不相关、有风险或属于编造，建议删除。
- `ask_user`：改写前还需要追问一个问题。

## 护栏（Guardrails）

- 不捏造数字。如果指标不明，追问证据或使用非数字结果表达。
- 不把团队成果写成个人成果，除非用户的具体角色已明确。
- 不声称精通用户只是"听说过"的工具。
- 不隐藏关键硬性条件缺口。
- 保持简历与面试故事库的一致性。

## 输出结构

1. JD 匹配诊断。
2. 最强可用证据。
3. 缺失证据与追问问题。
4. 改写建议。
5. 目标简历版本备注。
6. 面试追问风险提示。
