# 优势分类体系（Strength Taxonomy）

优势是有证据支撑的岗位信号，不是性格标签。

## 优势记录格式

```text
### 优势：<能力名>
- 分类（Classification）: fact | assumption | inference
- 证据（Evidence）: <真实经历或材料>
- 行为（Behavior）: <用户具体做了什么>
- 能力（Ability）: <可迁移能力>
- 岗位信号（Job signal）: <对哪些岗位有价值>
- 可信度（Confidence）: high | medium | low
- 缺失证据（Missing proof）: <还缺什么证据>
```

## 能力类别

| 类别 | 岗位信号 |
| --- | --- |
| 信息搜集与研究 | 能找到资料、比较来源、总结规律、做调研报告 |
| 结构化表达 | 能把混乱信息整理成文档、PPT、流程、表格 |
| 执行推进 | 能拆任务、跟进进度、协调资源、按时交付 |
| 数据与分析 | 会用表格、指标、实验、复盘解释问题 |
| 用户与市场理解 | 会观察用户反馈、评论、访谈、需求和痛点 |
| 内容与传播 | 会写作、剪辑、运营账号、组织活动、做增长 |
| 产品与流程意识 | 会画流程、拆需求、发现问题、优化体验 |
| 技术与工具使用 | 会编程、AI 工具、自动化、数据处理、低代码 |
| 沟通协作 | 能对齐目标、推动他人、处理冲突、复盘合作 |
| 学习迁移 | 能快速掌握新工具、新领域，并产出作品 |

## 证据质量等级

- **High（高）：** 有作品、链接、文件、截图、数据、第三方评价。
- **Medium（中）：** 有具体行动和结果，但缺少外部证据。
- **Low（低）：** 只有自我评价或模糊描述。

## 反模式（Anti-Patterns）

- 不把 MBTI、星座、性格词或"感觉适合"转化为职业结论。
- 不在缺乏证据和可补强行动说明的情况下，直接判定用户"不适合"某个方向。
- 不把一段薄弱经历过度解读为永久性的职业方向。
